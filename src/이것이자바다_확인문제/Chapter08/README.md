## 01. 인터페이스 (Interface)

### Q. 인터페이스에 대한 설명으로 틀린 것은 무엇입니까?

- [x] ➊ 인터페이스로 객체(인스턴스)를 생성할 수 있다.
- [ ] ➋ 인터페이스는 다형성의 주된 기술로 사용된다.
- [ ] ➌ 인터페이스를 구현한 객체는 인터페이스 타입으로 동일하게 사용할 수 있다.
- [ ] ➍ 인터페이스를 사용하면 객체 교체가 쉬워진다.

> **정답 : ➊**
>
> **해설 :**  
> 인터페이스는 객체 사용 방법을 정의한 설계도 역할을 한다.
>
> 따라서 `new` 연산자를 사용하여 직접 객체를 생성할 수 없다.
>
> 반드시 인터페이스를 구현한 클래스 객체를 생성해야 한다.

## 02. 인터페이스의 구성 멤버

### Q. 인터페이스의 구성 멤버에 대한 설명으로 틀린 것은 무엇입니까?

- [ ] ➊ 인터페이스는 인스턴스 필드가 없고 상수를 멤버로 가진다.
- [ ] ➋ 추상 메소드는 구현 클래스가 재정의해야 한다.
- [x] ➌ 디폴트 메소드는 구현 클래스에서 재정의할 수 없다.
- [ ] ➍ 정적 멤버는 구현 객체 없이 사용할 수 있다.

> **정답 : ➌**
>
> **해설 :**  
> 디폴트 메소드는 구현 클래스에서 오버라이딩할 수 있다.
>
> 구현 클래스는 기본 동작을 그대로 사용할 수도 있고,  
> 필요에 따라 재정의할 수도 있다.

## 03. 인터페이스 다형성 (Polymorphism)

### Q. 인터페이스 다형성에 대한 설명으로 틀린 것은 무엇입니까?

- [ ] ➊ 필드가 인터페이스 타입이면 다양한 구현 객체를 대입할 수 있다.
- [ ] ➋ 매개변수가 인터페이스 타입이면 다양한 구현 객체를 전달할 수 있다.
- [ ] ➌ 배열이 인터페이스 타입이면 다양한 구현 객체를 저장할 수 있다.
- [x] ➍ 구현 객체를 인터페이스 타입으로 변환하려면 강제 타입 변환을 해야 한다.

> **정답 : ➍**
>
> **해설 :**  
> 구현 객체를 인터페이스 타입 변수에 대입할 때는 자동 타입 변환이 발생한다.
>
> ```java
> RemoteControl rc = new Television();
> ```
>
> 강제 타입 변환은 인터페이스 타입을 다시 구현 클래스 타입으로 되돌릴 때 사용한다.

## 04. 인터페이스 매개변수의 다형성

### Q. 인터페이스 A를 B와 C가 구현하고, B를 상속해서 D 클래스, C를 상속해서 E 클래스를 만들었습니다.
### 다음 빈칸에 들어올 수 있는 것을 모두 선택하세요.

- [ ] ➊ new B()
- [x] ➋ new C()
- [ ] ➌ new D()
- [ ] ➍ new E()

> **정답 : ➋**
>
> **해설 :**  
> 클래스 `C`는 인터페이스 `A`를 구현한 클래스이다.
>
> 따라서 `new C()` 객체는 인터페이스 타입 `A`로 자동 타입 변환되어  
> 매개변수로 전달될 수 있다.
>
> 인터페이스를 사용하면 다양한 구현 객체를 동일한 방식으로 사용할 수 있으며,  
> 이를 다형성이라고 한다.
