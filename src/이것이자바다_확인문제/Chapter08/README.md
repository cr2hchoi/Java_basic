## 1. 인터페이스(Interface)
**Q. 인터페이스에 대한 설명으로 틀린 것은 무엇입니까?**
- [x] ➊ 인터페이스로 객체(인스턴스)를 생성할 수 있다.
- [ ] ➋ 인터페이스는 다형성의 주된 기술로 사용된다.
- [ ] ➌ 인터페이스를 구현한 객체는 인터페이스로 동일하게 사용할 수 있다.
- [ ] ➍ 인터페이스를 사용함으로써 객체 교체가 쉬워진다.

> **정답: ➊**
> **해설**: 인터페이스는 객체의 사용 방법을 정의한 '설계도'와 같은 개념으로, **추상 클래스와 마찬가지로 `new` 연산자를 사용하여 직접 객체(인스턴스)를 생성할 수 없다.** 반드시 인터페이스를 구현(implements)한 자식 클래스를 만든 뒤, 그 클래스를 통해 객체를 생성해야 한다.

### **나머지 보기 설명**
* **➋ 다형성의 기술**: 인터페이스는 여러 객체가 동일한 메소드 호출에 대해 각자 다른 방식으로 동작하게 만드는 다형성(Polymorphism)을 구현하는 핵심 수단이다.
* **➌ 동일한 사용**: 인터페이스를 구현한 클래스들은 해당 인터페이스 타입으로 변환될 수 있으며, 외부에서는 실제 어떤 객체인지 상관없이 인터페이스에 정의된 메소드들을 동일하게 호출할 수 있다.
* **➍ 객체 교체**: 프로그램이 클래스 타입이 아닌 인터페이스 타입을 사용하도록 설계하면, 나중에 구현 클래스를 다른 클래스로 교체하더라도 호출하는 쪽의 코드를 수정할 필요가 없어진다.

## 2. 인터페이스의 구성 멤버
**Q. 인터페이스의 구성 멤버에 대한 설명으로 틀린 것은 무엇입니까?**
- [ ] ➊ 인터페이스는 인스턴스 필드가 없고 상수를 멤버로 가진다.
- [ ] ➋ 추상 메소드는 구현 클래스가 재정의해야 하는 멤버이다.
- [x] ➌ 디폴트 메소드는 구현 클래스에서 재정의할 수 없다.
- [ ] ➍ 정적 멤버는 구현 객체가 없어도 사용할 수 있는 멤버이다.

### **💡 해설**
* **➌ 디폴트 메소드 재정의**: 인터페이스의 디폴트 메소드(`default method`)는 기본 구현 내용을 가지고 있는 메소드다. 구현 클래스에서 이 메소드를 그대로 사용할 수도 있지만, **필요에 따라 얼마든지 오버라이딩(재정의)하여 자신만의 실행 내용으로 바꿀 수 있다.** 자식 클래스에서 오버라이딩할 때는 `default` 키워드를 생략하고 구현해야 한다.

### **나머지 보기 설명**
* **➊ 상수 필드**: 인터페이스에 선언된 필드는 모두 public 정적 상수(`public static final`) 특성을 가진다. 객체마다 가질 수 있는 인스턴스 필드는 선언할 수 없다.
* **➋ 추상 메소드**: 인터페이스의 기본 목적으로, 선언부만 존재하며 구현 클래스가 반드시 오버라이딩하여 본체를 완성해야 하는 메소드다.
* **➍ 정적 멤버**: `static` 키워드가 붙은 정적 메소드는 인터페이스 이름으로 직접 접근하여 호출할 수 있으므로, 구현 객체가 없어도 바로 사용할 수 있다.

## 3. 인터페이스 다형성(Polymorphism)

**Q. 인터페이스 다형성에 대한 설명으로 틀린 것은 무엇입니까?**
- [ ] ➊ 필드가 인터페이스 타입일 경우 다양한 구현 객체를 대입할 수 있다.
- [ ] ➋ 매개변수가 인터페이스 타입일 경우 다양한 구현 객체를 대입할 수 있다.
- [ ] ➌ 배열이 인터페이스 타입일 경우 다양한 구현 객체를 저장할 수 있다.
- [x] ➍ 구현 객체를 인터페이스 타입으로 변환하려면 강제 타입 변환을 해야 한다.


### 💡 해설

구현 클래스로부터 생성된 객체가 인터페이스 타입의 변수에 대입될 때는 **자동 타입 변환(Promotion)**이 일어난다. 자식 클래스 객체가 부모 클래스 타입으로 자동 변환되는 것과 동일한 원리이므로, 강제 타입 변환(Casting)을 할 필요가 없다.
* **올바른 예시**: `RemoteControl rc = new Television();` (자동으로 변환되어 대입됨)


### **나머지 보기 분석**

* **➊ 필드의 다형성**: 클래스의 멤버 필드 타입을 인터페이스로 선언하면, 프로그램 실행 중에 해당 필드에 다양한 구현 객체를 교체해가며 대입할 수 있다.
* **➋ 매개변수의 다형성**: 메소드의 매개변수 타입을 인터페이스로 지정하면, 메소드를 호출할 때 그 인터페이스를 구현한 어떤 객체든 인자로 넘겨줄 수 있어 확장성이 넓어진다.
* **➌ 배열의 다형성**: 인터페이스 타입의 배열을 선언하면, 해당 인터페이스를 공통으로 구현한 서로 다른 클래스의 객체들을 하나의 배열 안에 묶어서 한 번에 관리할 수 있다.


### 📌 핵심 요약
* **자동 타입 변환**: `인터페이스 변수 = 구현객체;` 연산은 별도의 형변환 연산자 없이 자동으로 이루어진다.
* **강제 타입 변환이 쓰이는 경우**: 인터페이스 타입에 대입되어 있는 객체를 다시 원래의 구현 클래스 타입으로 되돌려 고유한 메소드를 쓰고 싶을 때만 강제 타입 변환(`(구현클래스) 인터페이스변수`)을 사용한다.

## 4. 인터페이스 매개변수의 다형성
**Q. 인터페이스 A를 B와 C가 구현하고 B를 상속해서 D 클래스를, C를 상속해서 E 클래스를 만들었습니다. 다음 빈칸에 들어올 수 있는 것을 모두 선택하세요.**


- [ ] ➊ new B()
- [x] ➋ new C()
- [ ] ➌ new D()
- [ ] ➍ new E()

---

### 💡 해설

1. **인터페이스 구현 관계**
    * 클래스 `C`는 인터페이스 `A`를 직접 구현(implements)한 클래스다.
    * 따라서 `new C()`로 생성된 객체는 부모 인터페이스인 `A` 타입의 매개변수(`A a`) 자리에 문제없이 대입될 수 있다.

2. **자동 타입 변환의 원리**
    * `void method(A a)` 구조에서 `C` 객체를 인자로 넘기면 `A a = new C();`가 성립되면서 자동 타입 변환이 일어난다.
    * 인터페이스 변수에 구현 객체를 대입하는 것은 자바의 가장 기본적인 다형성 메커니즘 중 하나다.

---

### 📌 핵심 요약 
* **인터페이스 타입 변환**: 구현 클래스는 해당 인터페이스 타입으로 변환되어 매개변수로 전달될 수 있다.
* **다형성 구현**: 인터페이스를 매개변수로 쓰면 구현체(`C` 등)의 종류에 따라 메소드 실행 결과가 다양해지는 효과를 얻는다.
