##  📝 Java 핵심 정리: 상속 (Inheritance)
### 1. 자바에서 상속의 정의와 특징 
**정의** 
>부모 클래스(Parent/Super Class)의 멤버(필드, 메서드)를 자식 클래스(Child/Sub Class)가 물려받는 것.
> - 자바에서는 물려받는다는 개념보단, 자식에 부모에게 상속을 요청하는 개념으로 이해하는 것이 더 좋다! 
- **키워드**: `extends` 사용.
- **특징**:
    - **단일 상속**: 자바는 클래스 간 다중 상속을 지원하지 않음 (부모는 하나만!).
    - **재사용성**: 공통 코드를 부모에게 몰아넣어 코드 중복을 제거함.
    - **가독성**: 클래스 간의 계층 관계를 파악하기 쉬워짐.

---

## 2. 핵심 키워드: super
- **super**: 자식 클래스에서 부모 객체에 접근할 때 사용하는 참조 변수.
- **super()**: 부모 클래스의 생성자를 호출함. (자식 생성자 첫 줄에서 자동 또는 수동 호출됨)
- **super.method()**: 오버라이딩으로 가려진 부모의 메서드를 호출할 때 사용.

---

## 3. 메서드 오버라이딩 (Method Overriding)
> "부모가 물려준 기능을 자식이 입맛에 맞게 고쳐 쓴다!"

- **성립 조건**:
    1. 메서드 이름, 매개변수 타입/개수, 리턴 타입이 부모와 동일해야 함.
    2. **접근 제한자**: 부모보다 좁은 범위로 변경 불가 (ex. 부모 public -> 자식 private 불가능).
    3. `@Override` 애노테이션 사용 권장 (실수 방지).



---

## 4. 다형성과 형변환 (Polymorphism)

### ⬆️ 업캐스팅 (Upcasting)
- **개념**: 자식 객체를 부모 타입 변수에 할당하는 것.
- **특징**: **자동 형변환**됨. 부모의 메서드만 호출 가능하지만, 자식이 **오버라이딩한 메서드가 우선 실행**됨.
- **예시**: `Animal myPet = new Dog();`

### ⬇️ 다운캐스팅 (Downcasting)
- **개념**: 업캐스팅된 부모 타입을 다시 자식 타입으로 변환하는 것.
- **특징**: **강제 형변환**이 필요하며, 자식 클래스 전용 메서드를 쓸 때 사용함.
- **안전장치**: `instanceof` 연산자로 실제 객체 타입을 확인하는 것이 필수.


---

## 5. 상속을 방지하는 final
- **final class**: 더 이상 상속할 수 없는 클래스 (자식을 가질 수 없음).
- **final method**: 더 이상 오버라이딩할 수 없는 메서드.

---

## 💡 상속 설계 Tip
1. **IS-A 관계**: "~은 ~의 일종이다"가 성립할 때만 상속을 사용할 것. (ex. Dog is an Animal)
2. **공통 분모**: 여러 클래스에서 공통으로 쓰는 필드와 메서드만 부모로 올릴 것.
3. **다형성 활용**: 매개변수나 배열 타입을 부모 클래스로 설정하면 코드가 유연해짐.
--- 


##  📝 Java 핵심 정리: 인터페이스 (Interface)

## 1. 인터페이스의 정의와 특징
- **정의**: 객체의 사용 방법을 정의한 일종의 **설계도(규격)**.
- **키워드**: `interface`로 선언, `implements`로 구현.
- **특징**:
    - **다중 구현**: 한 클래스가 여러 개의 인터페이스를 동시에 구현할 수 있음 (상속의 단일 상속 한계를 극복).
    - **완전 추상화**: 모든 메서드가 기본적으로 `public abstract`임 (구현부 `{}`가 없음).
    - **상수**: 모든 필드는 `public static final`임.



---

## 2. 왜 쓰는가? (장점)
1. **표준화**: 여러 개발자가 작업할 때 메서드 이름을 통일시켜줌 (규격화).
2. **관계 없는 클래스 간 공통 기능**: 상속 관계가 아닌 클래스들에게 동일한 기능을 부여함.
3. **결합도 낮추기 (Decoupling)**: 구현체가 바뀌어도 인터페이스를 사용하는 쪽의 코드는 수정할 필요가 없음.

---

## 3. 상속(extends) vs 인터페이스(implements)
> "상속은 혈통을 물려받는 것이고, 인터페이스는 기술을 배우는 것이다!"

| 구분 | **상속 (Inheritance)** | **인터페이스 (Interface)** |
| :--- | :--- | :--- |
| **키워드** | `extends` | `implements` |
| **관계** | IS-A (~은 ~의 일종이다) | HAS-A / CAN-DO (~을 할 수 있다) |
| **개수** | 단일 상속만 가능 | 다중 구현 가능 |
| **목적** | 부모의 기능을 확장/재사용 | 규격을 맞추고 기능을 강제함 |



---

## 4. 다형성과 인터페이스
- **업캐스팅**: 인터페이스 타입 변수에 구현 객체를 담을 수 있음.
    - 예: `RemoteControl rc = new Television();`
- **유연성**: `PetManager`에서 `Animal` 타입 대신 `Playable` 같은 인터페이스 타입을 쓰면, 동물뿐만 아니라 '로봇 강아지'도 놀아줄 수 있게 됨.

---

## 5. 인터페이스의 진화 (Java 8 이후)
원래는 추상 메서드만 가졌으나, 현재는 실행 코드를 가질 수 있는 기능이 추가됨.
- **default 메서드**: 인터페이스에서 기본 구현을 제공함. 자식 클래스에서 오버라이딩 안 해도 됨.
- **static 메서드**: 인터페이스 이름으로 바로 호출 가능한 공통 메서드.



---

## 💡 인터페이스 설계 Tip
1. **동사 위주의 네이밍**: 보통 `Runnable`, `Playable`, `Flyable` 등 "~할 수 있는" 능력 위주로 이름을 지음.
2. **강제성**: 인터페이스를 구현한 클래스는 **반드시** 모든 추상 메서드를 완성해야 함 (안 하면 컴파일 에러).
3. **다중 구현 활용**: `class Dog extends Animal implements Playable, Soundable` 처럼 상속과 인터페이스를 조합해 강력한 클래스를 만듦.
