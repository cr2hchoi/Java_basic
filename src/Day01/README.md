# 📝 Java 핵심 정리: 연산자 및 메서드

## 1. 연산자 (Operator)
### 1) 단항 연산자 (항이 1개)
- **증감 연산자**: `++`, `--` (변수 앞/뒤 위치에 따라 전위/후위 연산으로 나뉨).
- **부호 연산자**: `+`, `-` (양수/음수 표시).

### 2) 이항 연산자 (항이 2개)
- **산술 연산자**: `+`, `-`, `*`, `/`, `%`.
    - 정수끼리 계산하면 결과도 정수임.
    - 정수와 실수 계산 시 자동으로 실수형 변환되어 계산됨.
- **관계 연산자**: `>`, `<`, `>=`, `<=`, `==`, `!=`.
    - **instanceof**: 참조형 변수가 특정 클래스 타입인지 비교할 때 사용함 (중요).
- **논리 연산자**: `&&`(AND), `||`(OR), `!`(NOT).
- **복합 대입 연산자**: `+=`, `-=`, `*=`, `/=`, `%=`.

### 3) 삼항 연산자 (항이 3개)
- **형태**: `(조건식) ? A : B`
- **동작**: 조건식이 참이면 `A`, 거짓이면 `B`를 결과로 반환함.

## 2. 메서드 (Method)
- **정의**: 객체가 가진 동작을 의미함.
- **역할**: 필드 데이터 읽기/수정 및 외부와 상호작용하는 기능을 정의함.
- **장점**: 반복 작업을 묶어 코드 재사용성을 높인다.

### [함수 vs 메서드 차이점]
- **함수 (Function)**: 특정 동작 수행을 위한 독립적인 설계 코드 (어디에 속하지 않음).
- **메서드 (Method)**: 객체 기능을 구현하기 위해 **클래스 내부**에 구현된 것 (클래스에 종속적).

### [메서드 구분]
1. **정적 메서드 (Static Method)**:
    - 클래스에 고정된 메서드 
    - 객체 생성 없이 `클래스명.메서드명()`으로 호출
2. **인스턴스 메서드 (Instance Method)**:
    - 객체마다 개별적으로 존재하는 메서드
    - 반드시 `new`로 객체 생성 후 참조 변수(리모컨)를 통해 호출해야 한다.

# 🏗️  클래스의 구성 요소 

## 1. 필드 (Field)
- **정의**: 객체의 데이터(상태)가 저장되는 변수
- **특징**: 클래스 블록 내부 어디서든 사용 가능하며, 객체가 소멸되지 않는 한 유지됨.
- **비유**: 자동차의 '모델명', '현재 속도', '색상' 같은 데이터

## 2. 생성자 (Constructor)
- **정의**: `new` 연산자로 호출되는 특별한 블록
- **역할**: 객체 생성 시 초기화를 담당함 (인스턴스화의 관문).
- **특징**: 클래스 이름과 동일해야 하며, 리턴 타입이 없다.

## 3. 메서드 (Method)
- **정의**: 객체의 동작(기능)에 해당하는 블록
- **역할**: 필드 데이터를 읽거나 수정하고, 외부 객체와 데이터를 주고받는다
- **비유**: 자동차의 '가속하기', '브레이크 밟기', '기어 변속' 같은 동작임.

## 4. 내부 클래스 (Inner Class) - 선택적
- **정의**: 클래스 내부에 선언된 또 다른 클래스임.
- **역할**: 두 클래스가 서로 긴밀한 관계가 있을 때 선언하여 응집도를 높임.

## 💡 요약 및 팁
[연산자 및 매서드]
- 자바는 객체 지향 언어이므로 모든 함수는 클래스 안에 존재해야 하며, 이를 **메서드**라고 부른다. 
- **instanceof**는 다형성에서 부모 타입을 자식 타입으로 강제 형변환(다운캐스팅)하기 전, 안전한지 확인하는 용도로 필수 '_'
- 연산자 우선순위가 헷갈릴 땐 괄호`()`를 사용하는 게 제일 확실하다.

[클래스 요약]
- **필드**: 무엇을 가지고 있는가? (상태/데이터)
- **생성자**: 어떻게 태어나는가? (초기화)
- **메서드**: 무엇을 하는가? (기능/로직)

> **팁**: 클래스는 객체를 만들기 위한 **설계도**이고, 이 구성 요소들이 합쳐져 하나의 완성된 객체(인스턴스)가 된다.
